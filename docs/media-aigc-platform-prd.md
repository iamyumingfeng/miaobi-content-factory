# 妙笔内容工场 - 产品需求文档（PRD）

> 本文档定义了 妙笔内容工场的功能需求、业务流程和技术架构。

---

## 1. 产品概述

### 1.1 产品定位

**产品名称**：妙笔内容工场

**产品定位**：面向自媒体运营团队的 SaaS 平台，管理员可通过 AI 大模型实现爆款内容的复刻与深度批量二创，并将生成内容批量分发至创作者。

**目标平台**：小红书、抖音、公众号（后续可拓展视频号和视频内容）

### 1.2 核心角色

| 角色 | 说明 |
|------|------|
| 超级管理员 | 系统最高权限，仅 1 个，可管理所有创作管理员、创作者、设置和功能 |
| 创作管理员 | 可管理其模板、素材、内容生成、分发及其绑定的创作者 |
| 创作者 | 仅可查看自己被分发的内容，支持复制/下载，确认发布 |

### 1.3 产品形态

- **Web 管理后台**：全员可使用，基于 Vue 3 + Element Plus

## 2. 核心功能模块

### 2.1 用户与权限管理（RBAC）

#### 2.1.1 用户表结构（三表分离设计）

系统采用**三表分离设计**，通过 `UserBase` 抽象基类确保结构一致：

- **super_admin（超级管理员表）**：独立存储，owner_operator_id = NULL
- **operator（创作管理员表）**：独立存储，owner_operator_id = NULL
- **sub_user（创作者表）**：独立存储，owner_operator_id 必填

**共享字段**（UserBase抽象基类）：
- `userid`：用户ID（自动随机生成，登录用）
- `nickname`：管理员备注名（仅管理员可见）
- `display_name`：用户自定义昵称（用户自己可见）
- `hashed_password`：密码哈希
- `wechat_openid` / `wechat_unionid`：微信绑定
- `status`：online / offline / disabled
- `login_failure_count` / `locked_until`：登录安全控制
- `owner_operator_id`：所属创作管理员ID（数据隔离）
- `managed_by`：当前管理者ID（支持转移）

#### 2.1.2 登录方式

- **Web 端**：
  - 账号密码登录（userid + password）
  - 微信扫码登录（未绑定账号需先绑定）
  - JWT双Token机制：access_token（24h）+ refresh_token（7天）
- **小程序端**：微信授权登录

#### 2.1.3 个人设置功能

- 修改自定义昵称（display_name）
- 修改密码（需验证原密码）
- 查看/解绑微信绑定

#### 2.1.4 安全与合规

- 登录失败次数限制（连续 5 次失败锁定 15 分钟）
- 数据隔离：创作管理员仅能管理自己的资源（通过 owner_operator_id）
- JWT Token 时效控制：access_token 24小时，refresh_token 7天

### 2.2 模板管理

#### 2.2.1 模板四级层次结构

系统采用**四级层次结构**管理模板：

1. **模板平台（TemplatePlatform）**：如"小红书"、"抖音"、"微信公众号"
2. **模板分类（TemplateCategory）**：平台下的内容分类，如"美食"、"旅游"
3. **模板标签（TemplateTag）**：分类下的标签，用于进一步分类
4. **模板（Template）**：具体的内容模板

**设计说明**：
- 每个创作管理员可独立创建自己的平台、分类、标签体系
- 通过 `owner_operator_id` 实现完全数据隔离
- 超级管理员不能创建模板，必须使用创作管理员账号操作

#### 2.2.2 模板创建与编辑

- **基本信息**：名称、描述、所属平台/分类/标签、内容类型
- **提示词编辑**：
  - 提示词创意（description）
  - Prompt模板（支持变量占位符）
  - 变量定义（variables_json）
- **图片/视频配置**：
  - 图片数量（image_count）
  - 视频数量（video_count）
  - 图片尺寸比例（image_size_ratio）：1:1 / 4:3 / 16:9 / 3:4 / 9:16
  - 水印开关（add_watermark）
- **风格参考**：style_reference（风格场景参考描述）
- **平台适配**：platform_rules_json（平台适配规则）
- **模板状态**：enabled / disabled

#### 2.2.3 模板附件管理

- 支持上传图片、视频作为模板附件
- 附件信息：文件URL、文件名、大小、尺寸、时长
- 支持缩略图生成
- 附件排序（sort_order）

#### 2.2.4 模板操作

- 模板列表（支持平台/分类/标签筛选、关键词搜索）
- 卡片式/列表式双视图切换
- 模板复制编辑（original_template_id 追踪克隆关系）
- 批量操作：启用/禁用、删除

### 2.3 素材管理

#### 2.3.1 素材四级层次结构

系统采用**四级层次结构**管理素材：

1. **素材平台（MaterialPlatform）**：如"小红书"、"抖音"（素材库独立平台）
2. **素材分类（MaterialCategory）**：平台下的内容分类
3. **素材标签（MaterialTag）**：分类下的标签
4. **素材（Material）**：具体的素材内容

**素材库类型（library_type）**：
- **creation（创作库）**：用于内容生成的素材
- **benchmark（对标库）**：用于风格参考和对标的素材

#### 2.3.2 素材上传与创建

- **上传方式**：
  - 手动上传：文本、图片（多图）、视频
  - 链接导入：从URL导入（source_url）
  - 描述创建：纯描述性素材（source_type: upload / link / description）
- **素材信息**：
  - 标题（title）
  - 内容（content）
  - 主题（topic）
  - 文本内容（text_content）
  - 内容类型（content_type）：text / image_text / video_text / mix
- **素材目录规则**：符合COS目录规范，按 owner_operator_id 分桶

#### 2.3.3 素材附件管理

- 支持上传图片、视频作为素材附件
- 附件信息：文件URL、文件名、大小、尺寸、时长、缩略图
- 附件排序（sort_order）

#### 2.3.4 素材库管理（管理员）

- 素材列表：卡片式/列表式展示
- 素材筛选：按平台/分类/标签、素材库类型、时间、关键词
- 素材操作：查看、编辑、删除、收藏
- 批量操作：启用/禁用、删除
- 素材检索：按时间、关键词、类型筛选
- 素材操作：查看、编辑、删除
- 素材收藏

#### 2.3.3 素材查询（创作者）

- 仅只读查看被分发的内容
- 按时间、关键词检索
- 按状态筛选（已分发/已发布）

### 2.4 AIGC 内容批量生成

#### 2.4.1 生成任务创建

**步骤流程**：

1. **选择素材**：
   - 创作素材（material_id）：从创作库选择
   - 对标素材（benchmark_material_id）：从对标库选择（可选）
2. **选择模板**：可选择多个模板，编辑修改仅适用于当前任务
3. **配置模型**：
   - 文本模型：选择模型平台和具体模型
   - 图片模型：选择图片生成平台和模型
   - 模型选择模式：auto（自动） / manual（手动）
4. **配置并发数**：创作管理员可自定义最大并行数量（默认5）
5. **去重规则配置**：
   - **文案去重**：
     - 去重开关（dedup_enabled）
     - 相似度阈值（dedup_threshold，默认0.8）
     - 重试次数（dedup_retry_count，默认3）
     - 去重范围（dedup_scope）：subuser_history / operator_history
   - **图片去重**：
     - 去重开关（image_dedup_enabled）
     - 相似度阈值（image_dedup_threshold，默认0.8）
     - 重试次数（image_dedup_retry_count，默认3）
     - 去重范围（image_dedup_scope）
6. **对标配置**：
   - 文案对标开关（benchmark_text_enabled）
   - 图片对标开关（benchmark_image_enabled）
   - 图片参考选项（benchmark_image_reference_options）
   - 对标图角色配置（benchmark_image_roles_json）
7. **创意种子配置**：
   - 开头模式（opening_seed_id）：auto（自动）或指定种子ID
   - 情感基调（emotion_seed_id）：auto（自动）或指定种子ID
   - 结尾模式（ending_seed_id）：auto（自动）或指定种子ID
   - 说明：创意种子用于强制差异化，避免生成内容趋同
8. **勾选目标创作者**：支持按标签筛选
9. **确认提交**

#### 2.4.2 异步生成引擎

- **子任务分解**：按（创作者 × 模板）组合分解为对应子任务
- **个性化生成**：依据模板、素材、创作者定位综合生成
- **输入快照**：记录模板、素材、创作者信息的完整快照
- **执行日志**：记录每个流水线节点的执行情况（generation_item_execution_log）
- **并行处理**：支持自定义并行数量
- **并发控制**：各模型平台设置最大并发数
- **自动模型切换**：遇并发限制自动切换其他平台（auto模式）
- **失败重试**：自动重试 3 次（指数退避），超过放入失败列表
- **手动重试**：支持单个或批量重试

#### 2.4.3 去重检测机制

**文案去重流程**：
1. 生成文案后，调用 embedding 模型生成向量
2. 从 content_embedding 查询历史文案向量（根据 dedup_scope）
3. 计算相似度（cosine similarity）
4. 如果相似度 > dedup_threshold，重试生成（最多 dedup_retry_count 次）
5. 保存新的文案向量到 content_embedding

**图片去重流程**：
1. 生成图片后，调用 embedding 模型生成图片向量
2. 从 content_embedding 查询历史图片向量（根据 image_dedup_scope）
3. 计算每张图片的相似度
4. 如果相似度 > image_dedup_threshold，重试生成（最多 image_dedup_retry_count 次）
5. 保存新的图片向量到 content_embedding（每张图片一条记录）

#### 2.4.4 生成进度与状态

- **任务状态概览**：
  - 总数、队列中、生成中、已完成、失败、已暂停
  - 已分发、待发布、已发布数量和比例
- **子任务状态流转**：
  - queued（排队中）→ generating（生成中）→ completed / failed / paused
  - distribution_status: draft → ready → distributed → pending_publish → published
- **批量操作**：暂停/继续、失败重试
- **实时监控**：WebSocket 推送进度

#### 2.4.5 生成结果预览与管理

- 文本直接查看，图片/视频在线预览
- 查看最终提示词（final_prompt）、系统提示词、用户提示词
- 查看去重检测结果（相似度、引用内容）
- 单条内容重新生成
- 批量删除生成素材
- 过期清理策略（7/15/30/60/90 天）

### 2.8 创意种子库管理

#### 2.8.1 种子类型

系统支持三种创意种子类型：

| 种子类型 | 说明 | 示例 |
|---------|------|------|
| 开头模式 | 控制文案开头的风格 | 反常识、自嘲、悬念、痛点场景 |
| 情感基调 | 控制文案的情感表达 | 轻松吐槽、真诚分享、犹豫观望 |
| 结尾模式 | 控制文案的收尾方式 | 求建议、留悬念、共鸣式 |

#### 2.8.2 种子管理功能

- **系统预置种子**：系统预置高质量种子，创作管理员可直接使用
- **自定义种子**：创作管理员可创建自己的创意种子
- **种子编辑**：可修改种子名称、模板、说明、禁止模式等
- **种子状态**：支持启用/禁用，禁用后不会出现在生成任务中
- **品类适配**：支持按品类筛选种子（美妆、美食、数码等）

#### 2.8.3 种子组合机制

- 每次生成任务从三种类型中各选一个种子组合
- 组合格式：`开头模式 + 情感基调 + 结尾模式`
- 支持指定种子ID或自动随机选择
- 系统预置组合数：7 × 7 × 6 = 294 种

#### 2.8.4 差异化效果

- **降低调用次数**：通过强制差异化框架，减少重试次数
- **提升内容质量**：避免AI生成趋同性内容
- **降低相似度**：从源头确保内容差异化

## 4. 核心数据模型

### 4.1 用户表 (all_user)

采用统一用户表设计，通过 `role` 字段区分角色：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键，自增 |
| userid | string | 用户 ID（自动生成，唯一） |
| nickname | string | 管理员备注名 |
| display_name | string | 用户自定义昵称 |
| password_hash | string | 密码哈希 |
| role | enum | super_admin / operator / sub_user |
| wechat_openid | string | 微信 OpenID |
| owner_admin_id | bigint | 所属创作管理员 ID（数据隔离） |
| status | enum | online / offline / disabled |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

**索引设计**：

- `idx_userid`：userid 唯一索引
- `idx_role`：role 普通索引
- `idx_owner_admin`：owner_admin_id 普通索引

### 4.2 核心业务表

| 表名 | 说明 |
|------|------|
| user_tag / user_tag_rel | 用户标签管理 |
| template_platform | 模板平台分类 |
| template | 内容模板（含变量占位符） |
| material | 素材（文本/图片/视频） |
| material_attachment | 素材附件 |
| generation_task | 生成任务（包含完整进度统计） |
| generation_item | 生成明细/子任务 |
| distribution | 分发记录 |
| model_config | 模型配置 |
| operation_log | 操作日志 |

### 4.3 数据隔离设计

- **多租户隔离**：所有核心表包含 `owner_admin_id` 字段
- **复合索引**：
  - `idx_owner_operator_date(owner_admin_id, created_at)`
  - `idx_subuser_item(sub_user_id, status)`
- **表分区策略**：
  - `generation_item` 按周分区
  - `generation_task` 按月分区

## 5. 异步任务流程

### 5.1 任务提交流程

```
1. 管理员提交生成任务
   ↓
2. 创建 generation_task 记录（status=pending）
   ↓
3. 为每个（创作者，模板）组合创建 generation_item
   ↓
4. Celery Worker 从队列拉取子任务
   ↓
5. 检查模型平台并发限制（Redis 计数器）
   ↓
6. 遇并发限制 → 自动切换平台 或 放回队列重试
   ↓
7. 调用 AI 模型生成内容
   ↓
8. 更新 generation_item 状态
   ↓
9. WebSocket 推送进度到前端
```

### 5.2 并发控制机制

- **模型并发限制**：各平台设置最大并发数
- **自适应调节**：遇限流自动限流当前最大并行
- **失败重试**：指数退避（30s → 60s → 120s）
- **手动重试**：支持单个/批量重试
